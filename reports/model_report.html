The build, in six moves

Every stage below runs on the Cleveland-style raw match data — cleaned, filtered, reshaped, and engineered without any ML library touching the model itself.

01

Clean

49,487 international matches kept, 11 rows with no recorded score dropped.

02

Filter

Narrowed to 1,046 FIFA World Cup finals matches only.

03

Reshape

One row per team per match — 2,092 team-match records.

04

Engineer

Rolling form and goals, host flag, and FIFA rank differential — all leakage-checked.

05

Train

Batch gradient descent on cross-entropy loss, converged in 265 steps.

06

Validate

5-fold cross-validation, then checked against scikit-learn.

The lineup: what actually predicts a win

Trained weight for each feature, standardised so magnitudes are directly comparable. One factor towers over the rest.

Trained weights — rank differential 0.774, host flag 0.180, goals scored 0.155, recent form 0.086, goals conceded 0.075.
• Note on goals conceded. Its coefficient flipped from negative (without ranking data) to a small positive once FIFA rank was added. The two features carry a real correlation of −0.293 — weaker teams both concede more and rank worse — so rank differential absorbs most of that signal. L2 regularisation was tested and didn't change this, confirming it's overlap between features, not overfitting.

Turning the dial: L2 regularisation

Swept the penalty strength from 0 to 10 under the same 5-fold setup, watching mean AUC move.

L2 sweep: lambda 0 to 10, AUC stays flat between 0.7266 and 0.7271.
• Flat, and that's the correct result. With 5 features and 1,156 matches, there's little room for the model to overfit — so regularisation has almost nothing to correct. Knowing when a technique doesn't help is as useful as knowing when it does.

Known limits

Ranked matches only go back to 1992, so the final training set is 1,156 rows. Head-to-head win rate was dropped — 65.8% of pairings had no prior meeting. A 5-feature linear model still misses over half of true wins on average.

Built with

No scikit-learn in the model itself — sigmoid, cross-entropy loss, and gradient descent are hand-derived and hand-coded.
