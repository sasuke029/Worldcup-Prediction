# World Cup win predictor — logistic regression from scratch

A binary classifier predicting FIFA World Cup match outcomes (win vs.
not-win), implemented from first principles in NumPy. No scikit-learn,
no ML libraries anywhere inside the model itself — sigmoid, cross-entropy
loss, and gradient descent are all hand-derived and hand-coded.

Includes a full data pipeline, from-scratch training with cross-validation,
validation against scikit-learn, and two frontends: a results dashboard
and an interactive match predictor.

**[Full methodology and math &rarr; docs/METHODOLOGY.md](docs/METHODOLOGY.md)**

## Results

Trained on 1,129 FIFA World Cup team-match records (the period with FIFA
ranking data available).

| Metric | v1 &mdash; form &amp; goals only | v2 &mdash; + FIFA rank differential |
|---|---|---|
| Accuracy | 0.632 | 0.724 |
| Precision | 0.615 | 0.712 |
| Recall | 0.154 | 0.483 |
| F1 | 0.246 | 0.575 |
| ROC-AUC | 0.622 | 0.788 |

**5-fold cross-validated AUC: 0.730 &plusmn; 0.040** (the more honest number
&mdash; a single train/test split can flatter a model).

**Validated against scikit-learn's `LogisticRegression`:** trained weights
differ by no more than ~0.02 per feature, confirming the from-scratch
gradient descent converges to the same solution as a production optimiser.

## What predicts a win

Trained feature weights, largest to smallest:

| Feature | Weight | Meaning |
|---|---|---|
| FIFA rank differential | 0.77 | By far the strongest signal &mdash; the better-ranked team is much more likely to win |
| Host nation | 0.18 | Real, measurable home-tournament advantage |
| Goals scored (avg, last 5) | 0.16 | Recent attacking form |
| Recent win rate (last 5) | 0.09 | Recent overall form |
| Goals conceded (avg, last 5) | 0.07 | Weakest and least stable &mdash; overlaps with rank differential (correlation &minus;0.29) |

## Data

Download and place in `data/raw/`:

1. [International football results, 1872&ndash;2017](https://www.kaggle.com/datasets/martj42/international-football-results-from-1872-to-2017) &rarr; `results.csv`
2. FIFA World Ranking history (search Kaggle "FIFA World Ranking") &rarr; `fifa_ranking.csv`

## Run

```bash
python -m src.train
```

Builds the dataset, trains the model, prints held-out and cross-validated
metrics, and exports `reports/model_weights.json` with the exact trained
weights and feature scaler.

## Tests

```bash
python -m pytest tests/
```

4 sanity tests: sigmoid output bounds, correct convergence on separable
data, metric correctness on known cases, and AUC &asymp; 0.5 on random
predictions.

## Known limitations

- FIFA rankings only exist from December 1992 onward, so early World
  Cups are excluded from training &mdash; final dataset is ~1,100&ndash;1,150 rows
  depending on the ranking snapshot used.
- Head-to-head win rate was engineered and dropped: 65.8% of team
  pairings had no prior World Cup meeting, making it too sparse to use.
- A 5-feature linear model has a real ceiling &mdash; recall stays around
  0.46&ndash;0.48 under cross-validation, meaning it still misses roughly
  half of true wins.
- The predictor's draw probability is a derived residual
  (`1 − P(win) − P(opponent win)`), not a separately trained outcome.
  A proper three-way (win/draw/loss) model would need a multinomial
  extension &mdash; see `docs/METHODOLOGY.md`, Section 9.
- L2 regularisation was tested across &lambda; &isin; [0, 10] and made
  negligible difference, consistent with a model that isn't overfitting
  given its low feature-to-sample ratio.
